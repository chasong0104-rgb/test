# 원여고 분실물 지도 — 제작 과정

## 1. 프로젝트 소개

원여고등학교 학생들이 교내에서 **잃어버린 물건을 쉽게 찾을 수 있도록** 만든 웹앱입니다.
학교의 **실제 층별 도면(1~5층)** 위에서 분실물이 어디에 있는지 확인하고, 직접 등록할 수도 있습니다.
스마트폰에서 **앱처럼 설치**해서 쓸 수 있습니다(PWA).

- 실제 서비스 주소: https://chasong0104-rgb.github.io/test/

## 2. 사용한 기술

| 구분 | 사용 기술 |
|------|-----------|
| 화면(프론트엔드) | HTML · CSS · JavaScript (프레임워크·빌드도구 없이 순수 코드) |
| 데이터베이스/서버 | **Supabase** (PostgreSQL 데이터베이스 + 로그인 + 실시간) |
| 배포 | **GitHub + GitHub Pages** (자동 배포) |
| 앱 기능 | PWA (서비스워커, 매니페스트로 설치·오프라인 지원) |

## 3. 개발 과정

### (1) 처음엔 Firebase로 시작
처음에는 구글의 **Firebase**(Firestore 데이터베이스 + 로그인)로 만들어 배포했습니다.
그런데 사용 중 **두 가지 문제**를 발견했습니다.

1. **등록자 본인만 분실물이 보이고, 다른 사람은 볼 수 없음**
   → 원인: 데이터베이스 보안 규칙이 "본인 것만 읽기"로 되어 있었음
2. **로그인한 사람이 분실물을 등록해도 포인트가 쌓이지 않음**
   → 원인: 포인트를 적립하는 코드가 아예 없었음

### (2) Firebase → Supabase 로 전환
더 유연한 데이터베이스(SQL)와 세밀한 보안 설정을 위해 **Supabase로 옮기기로** 결정했습니다.
데이터베이스 구조를 직접 설계하고 다루면서 작업을 진행했습니다.

### (3) 데이터베이스 설계 (Supabase / PostgreSQL)
표(테이블) 2개를 만들었습니다.

- **items** — 분실물/찾는 물건 게시물
  (제목, 카테고리, 사진, 발견 위치, 현재 위치, 상태, 게시물 유형 등)
- **profiles** — 회원 정보와 포인트

그리고 다음을 설정했습니다.

- **보안 규칙(RLS)**: 분실물은 **누구나 조회 가능**하도록 변경 → *문제 1 해결*.
  프로필은 본인 것만 수정 가능하도록 제한.
- **자동 처리(트리거)**
  - 회원가입하면 프로필이 **자동 생성**됨
  - 로그인 사용자가 분실물을 등록하면 **자동으로 +10P 적립** → *문제 2 해결*
- **실시간 반영(Realtime)**: 한 사람이 글을 올리면 접속 중인 **모든 사람 화면에 즉시** 반영.

### (4) 화면(클라이언트) 다시 작성
기존 Firebase 코드를 걷어내고 **Supabase 코드로 교체**했습니다.
로그인은 학생들이 이메일 없이 **아이디 + 비밀번호**만으로 가입할 수 있게 만들었습니다
(내부적으로는 이메일 형식으로 변환해 저장).

### (5) 주요 기능 구현
- **지도**: 실제 원여고 5개 층 도면 위에 방마다 핀 표시. 층/장소를 선택하면 그곳의 분실물 조회
- **위치 색 구분**: 처음 발견된 위치(파랑) vs 현재 보관 위치(주황)를 다른 색으로
- **검색**: 장소·카테고리·물건 이름으로 찾기
- **정렬**: 최신순 / 오래된순
- **포인트 & 랭킹**: 로그인 등록 시 +10P, 매월 초기화, 전체 회원 중 상위 3위 랭킹
- **찾습니다**: 물건을 잃어버린 사람이 직접 찾는 글을 올리는 전용 페이지
- **보관함**: 주인을 찾았거나 등록 30일이 지난 글을 따로 모아 보관
- **테마**: 원여(핑크 벚꽃) / 빈티지(세피아) 전환

### (6) 배포
- 코드를 **GitHub 저장소**에 올리고 **GitHub Pages**로 배포
- 이후에는 코드를 수정해 GitHub에 올리면(push) **사이트에 자동으로 반영**됨

## 4. 어려웠던 점과 해결

| 어려웠던 점 | 해결 방법 |
|-------------|-----------|
| 보안 규칙 때문에 데이터가 다른 사람에게 안 보임 | Supabase RLS 정책으로 "누구나 조회 가능"하게 수정 |
| 실제 도면 위에 핀을 정확히 배치 | 도면 이미지를 분석해 각 방을 백분율(%) 좌표로 배치 |
| 층 순서가 뒤바뀌어 있던 문제 | 데이터에서 층 번호를 올바르게 재배치 |
| 도면 이미지에 인쇄된 "1F/2F" 글자 | 이미지의 해당 부분을 배경색으로 덮어 제거 |
| 포인트를 클라이언트가 조작할 수 있는 위험 | 포인트 적립을 데이터베이스 트리거(서버)에서 처리 |
